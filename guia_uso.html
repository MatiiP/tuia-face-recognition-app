🐳 Guía de Instalación y Uso

Paso a paso para configurar, ejecutar y operar el sistema de reconocimiento facial con Docker.

Requisitos Previos

  • Docker Desktop — Descargar aquí
  • Git — Para clonar el repositorio
  • Al menos 8 GB de RAM disponibles para Docker
Nota: No necesitás tener Python instalado localmente. Todo corre dentro de los contenedores Docker.

1 Clonar el Repositorio

git clone https://github.com/tu-usuario/tuia-face-recognition-app.git
cd tuia-face-recognition-app

2 Levantar los Servicios con Docker

Desde la raíz del proyecto, ejecutá:

docker compose up --build -d

Este comando construye las imágenes e inicia 4 servicios:

ServicioPuertoURLFunción
backend8000http://localhost:8000API REST (FastAPI)
frontend8080http://localhost:8080Interfaz gráfica (Gradio)
postgres5432—Base de datos vectorial (pgvector)
jupyter8888http://localhost:8888JupyterLab

Verificar que todo esté corriendo

docker compose ps

Deberías ver los 4 contenedores con estado Up.

3 Abrir la Interfaz Gráfica (Gradio)

Abrí tu navegador e ingresá a: http://localhost:8080

La interfaz tiene 3 pestañas:

Pestaña "Predecir"

  1. Subí una imagen con uno o más rostros.
  2. Hacé clic en "Iniciar predicción".
  3. Copiá el job_id que aparece.
  4. Hacé clic en "Consultar resultado de este job".
  5. Se muestra la imagen con bounding boxes, keypoints y la identidad reconocida.

Pestaña "Registrar identidad"

  1. Escribí el nombre de la persona.
  2. Subí una foto donde se vea un solo rostro claro.
  3. Hacé clic en "Registrar".
  4. La identidad se guarda en la base de datos PostgreSQL.

Pestaña "Estado y resultados"

  1. Pegá un job_id.
  2. Hacé clic en "Consultar".
  3. Podés ver el JSON, la imagen anotada y links de descarga.

4 Poblar la Base de Datos

Para cargar automáticamente 2 imágenes de cada persona (excepto Valentino):

docker compose run --rm --entrypoint python jupyter seed_db.py

Salida esperada:

Registrando ambrogi — ambrogi_02.png... ✓
Registrando gianfranco — gianfranco_01.png... ✓
...
⏭ Saltando a valentino (excluido intencionalmente)

Seed finalizado: 12 identidades registradas con éxito.

5 Entrenar (o Re-entrenar) el Modelo

5.1 — Preparar el Dataset

Las imágenes deben estar en Dataset/ organizadas por persona:

Dataset/ ├── ambrogi/ (todas las imágenes — prueba de desbalance) ├── gianfranco/ (hasta 10 imágenes) ├── gianluca/ (hasta 10 imágenes) ├── lucas/ (hasta 10 imágenes) ├── matias/ (hasta 10 imágenes) ├── roberto/ (hasta 10 imágenes) └── valentino/ (excluido automáticamente del entrenamiento)

5.2 — Ejecutar el Entrenamiento

docker compose run --rm --entrypoint python jupyter train.py

El script entrena un ResNet-18 durante 50 épocas y exporta el modelo a models/face_detection.pth.

5.3 — Parámetros configurables

Podés editar estos valores al inicio de train.py:

ParámetroDefaultDescripción
EPOCHS50Cantidad de épocas
BATCH_SIZE8Tamaño del batch
LR1e-4Learning rate (Adam)
FACE_SIZE112Tamaño del rostro alineado
EMBEDDING_SIZE512Dimensión del embedding

5.4 — Después de entrenar

Importante: Después de re-entrenar el modelo, debés reiniciar el backend, limpiar la base de datos y volver a poblarla. Los embeddings del modelo viejo no son compatibles con el nuevo. 
# 1. Reiniciar el backend para que cargue el modelo nuevo
docker compose restart backend

# 2. Limpiar la base de datos
docker compose exec postgres psql -U faces_user -d faces -c "DELETE FROM embeddings;"

# 3. Re-poblar con el modelo nuevo
docker compose run --rm --entrypoint python jupyter seed_db.py

6 Generar Gráficos de Evaluación (PCA / t-SNE)

docker compose run --rm --entrypoint python jupyter evaluate.py

Genera:

  • output/pca.png — Gráfico PCA de todos los embeddings
  • output/tsne.png — Gráfico t-SNE (Valentino marcado con ★)
  • output/evaluation_report.json — Similitudes y vecinos más cercanos

7 Consultar la Base de Datos

# Abrir consola SQL interactiva
docker compose exec postgres psql -U faces_user -d faces

# Consultas útiles:
SELECT name FROM embeddings;          -- Ver identidades
SELECT count(*) FROM embeddings;      -- Contar registros
\d embeddings;                        -- Ver estructura
\q                                    -- Salir
